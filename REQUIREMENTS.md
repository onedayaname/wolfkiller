# 狼人杀现场游戏辅助工具 - 需求文档

## 1. 项目概述

### 1.1 项目目标
开发一个基于Web的狼人杀现场游戏辅助工具，帮助法官高效地管理现场游戏，包括角色分配、身份查看、游戏进程记录等功能。

### 1.2 目标用户
- 狼人杀游戏法官（主要用户）
- 现场玩家（次要用户，仅用于查看身份）

### 1.3 使用场景
线下狼人杀游戏现场，法官使用手机访问该工具，辅助游戏进行。

## 2. 功能需求

### 2.1 游戏配置模块

#### 2.1.1 玩家数量设置
- 输入现场玩家数量（6-20人）
- 系统自动根据玩家数量推荐角色配置
- 用户可手动调整各角色数量

#### 2.1.2 角色配置
- 支持的角色类型：
  - 狼人
  - 村民
  - 预言家
  - 女巫
  - 猎人
  - 守卫
  - 白痴
  - 骑士
  - 白狼王
  - 石像鬼
  - 恶魔
  - 狼美人
- 完全自定义角色数量
- 角色数量总和必须等于玩家数量
- 验证配置合法性（如：至少1个狼人、至少1个神职、至少1个村民）

#### 2.1.3 游戏开始
- 确认配置后随机分配角色
- 生成玩家列表（玩家1、玩家2...或自定义玩家名）

### 2.2 身份查看模块

#### 2.2.1 身份查看流程
- 按顺序让玩家查看身份
- 每次只显示一个玩家的身份
- 防止身份暴露机制：
  - 身份显示后需要手动隐藏
  - 隐藏前无法查看下一个玩家身份
  - 显示身份时使用遮罩效果
  - 提供明确的操作提示

#### 2.2.2 身份显示
- 显示玩家编号/名称
- 显示角色名称
- 显示角色技能说明（可选）
- 使用醒目的视觉设计区分阵营（好人/坏人）

### 2.3 游戏进程管理模块

#### 2.3.1 玩家状态管理
- 记录玩家状态：存活/死亡
- 记录死亡时间（第几轮、白天/黑夜）
- 记录死亡原因（投票/狼刀/技能）
- 可修改玩家状态（误操作可撤销）

#### 2.3.2 技能使用记录
- 记录各角色技能使用情况
- 简单记录：仅记录是否使用
- 支持的技能记录：
  - 预言家查验
  - 女巫救人/毒人
  - 猎人开枪
  - 守卫守护
  - 骑士决斗
  - 白狼王自爆
  - 狼美人魅惑
- 记录技能目标（玩家编号）

#### 2.3.3 游戏轮次管理
- 记录当前游戏轮次
- 区分白天和黑夜
- 显示当前阶段（天黑请闭眼、天亮了、投票等）

### 2.4 胜负判断模块

#### 2.4.1 胜负判断规则
- 屠边规则（默认）：
  - 狼人胜利：所有神职死亡 或 所有村民死亡
  - 好人胜利：所有狼人死亡
- 屠城规则（可选）：
  - 狼人胜利：所有好人死亡
  - 好人胜利：所有狼人死亡

#### 2.4.2 自动判断
- 根据玩家死亡情况实时判断胜负
- 达成胜利条件时显示提示
- **不自动结束游戏**，需要法官确认

#### 2.4.3 游戏结束
- 法官确认后结束游戏
- 显示胜利方
- 显示游戏统计信息（存活玩家、死亡玩家、技能使用情况）

## 3. 非功能需求

### 3.1 性能要求
- 页面加载时间 < 2秒
- 操作响应时间 < 500ms
- 动画流畅，帧率 > 60fps

### 3.2 兼容性要求
- 支持主流移动浏览器（Chrome、Safari、Firefox）
- 适配不同屏幕尺寸（手机、平板）
- 支持横屏和竖屏

### 3.3 用户体验要求
- 界面美观，视觉效果好
- 操作简单直观
- 提供清晰的操作提示
- 防止误操作

### 3.4 安全性要求
- 不保存游戏数据，刷新页面后数据清空
- 防止身份泄露
- 防止未授权访问

## 4. 技术栈选择

### 4.1 前端框架
- **Next.js 14** (App Router)
- **React 18**
- **TypeScript**

### 4.2 UI组件库
- **shadcn/ui** (美观、现代、可定制)
- **Tailwind CSS** (样式框架)
- **Framer Motion** (动画库)

### 4.3 状态管理
- **Zustand** (轻量级状态管理)
- **React Context** (全局状态)

### 4.4 部署方案
- **Vercel** (云端部署)
- 支持HTTPS
- 自动CI/CD

## 5. 用户界面设计

### 5.1 设计风格
- 暗色主题（符合狼人杀氛围）
- 使用狼人杀相关视觉元素（月亮、狼、眼睛等）
- 渐变色、阴影效果
- 流畅的过渡动画

### 5.2 主要页面

#### 5.2.1 首页
- 游戏标题
- 开始新游戏按钮
- 游戏规则说明（可选）

#### 5.2.2 游戏配置页
- 玩家数量输入
- 角色配置列表（可调整数量）
- 开始游戏按钮

#### 5.2.3 身份查看页
- 当前查看的玩家信息
- 身份显示区域（带遮罩）
- 下一个玩家按钮（隐藏后可用）
- 进度指示器

#### 5.2.4 游戏进行页
- 玩家列表（显示状态）
- 当前轮次和阶段
- 操作按钮：
  - 标记玩家死亡
  - 记录技能使用
  - 下一轮
  - 结束游戏

#### 5.2.5 游戏结束页
- 胜利方显示
- 游戏统计
- 开始新游戏按钮

### 5.3 响应式设计
- 优先考虑手机端体验
- 使用相对单位（rem、vw、vh）
- 触摸友好的按钮尺寸（最小44px）
- 避免横向滚动

## 6. 游戏流程

### 6.1 完整流程
1. 法官打开应用
2. 配置游戏（玩家数量、角色配置）
3. 开始游戏
4. 依次让玩家查看身份（防止泄露）
5. 游戏开始，进入天黑
6. 记录狼刀、技能使用
7. 天亮，公布死讯
8. 记录投票结果
9. 重复步骤6-8
10. 达成胜利条件，系统提示
11. 法官确认，游戏结束
12. 显示结果，开始新游戏

### 6.2 异常处理
- 误操作可撤销
- 网络中断不影响当前游戏（本地状态）
- 刷新页面会丢失数据（符合需求）

## 7. 数据结构设计

### 7.1 游戏状态
```typescript
interface GameState {
  gameId: string;
  playerCount: number;
  players: Player[];
  currentRound: number;
  currentPhase: 'night' | 'day';
  isGameOver: boolean;
  winner: 'wolf' | 'good' | null;
  gameRule: '屠边' | '屠城';
}
```

### 7.2 玩家信息
```typescript
interface Player {
  id: number;
  name: string;
  role: Role;
  status: 'alive' | 'dead';
  deathInfo?: {
    round: number;
    phase: 'night' | 'day';
    cause: 'wolf' | 'vote' | 'witch' | 'hunter' | 'knight' | 'guard' | 'other';
  };
}
```

### 7.3 角色信息
```typescript
interface Role {
  name: string;
  type: 'wolf' | 'god' | 'villager';
  skill?: string;
}
```

### 7.4 技能使用记录
```typescript
interface SkillUsage {
  id: string;
  round: number;
  phase: 'night' | 'day';
  playerId: number;
  role: string;
  skill: string;
  targetId?: number;
}
```

## 8. 开发计划

### 8.1 开发阶段
1. **阶段一**：项目搭建和基础UI（2天）
   - 初始化Next.js项目
   - 配置shadcn/ui和Tailwind
   - 创建基础页面结构

2. **阶段二**：游戏配置模块（1天）
   - 玩家数量设置
   - 角色配置
   - 配置验证

3. **阶段三**：身份查看模块（1天）
   - 身份分配逻辑
   - 身份显示界面
   - 防泄露机制

4. **阶段四**：游戏进程管理（2天）
   - 玩家状态管理
   - 技能使用记录
   - 轮次管理

5. **阶段五**：胜负判断（1天）
   - 胜负判断逻辑
   - 游戏结束流程

6. **阶段六**：UI优化和测试（1天）
   - 动画效果
   - 响应式适配
   - 功能测试

### 8.2 总计
- 预计开发时间：8天
- 测试和优化：2天
- 总计：10天
